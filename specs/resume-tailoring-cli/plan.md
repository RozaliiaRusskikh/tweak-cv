# Implementation Plan: Resume Tailoring CLI

**Branch**: `resume-tailoring-cli` | **Date**: 2026-06-01 | **Spec**: [spec.md](./spec.md)

## Summary

Build a personal CLI tool that takes a job description, uses a LangGraph state machine (Gemini 2.0/2.5 Flash via `langchain-google-genai`) to tailor a base resume, posts the result to Slack with Approve/Edit/Reject buttons, and exports an approved PDF via WeasyPrint. All runs are traced and scored in Langfuse (free Hobby tier). The Slack bot is the only UI — no web frontend.

---

## Technical Context

**Language/Version**: Python 3.13

**Primary Dependencies**:
- `langgraph` — workflow orchestration, HITL via `interrupt()`, SqliteSaver checkpointer
- `langchain-google-genai` — Gemini LLM calls with Pydantic structured output (`.with_structured_output()`)
- `langfuse` — observability, prompt management (primary), tracing & scoring
- `slack-bolt` + `fastapi` + `uvicorn` — Slack bot action handlers + webhook server
- `sqlalchemy` — SQLite ORM (jobs, resume_versions)
- `weasyprint` + `jinja2` — HTML/CSS → PDF export
- `python-dotenv`, `pydantic`, `click`, `loguru`

**Storage**: SQLite, two files — `tailorcv.db` (app data) + `checkpoints.db` (LangGraph SqliteSaver), both under `/app/data/`

**Testing**: Phase-by-phase smoke tests; `evals/run_evals.py` for offline prompt benchmarking

**Target Platform**: Linux container (Docker required for WeasyPrint system font/layout libraries)

**Project Type**: Persistent webhook server (`uvicorn tweakcv.slack_handler:app` runs in Docker); a JD posted to the Slack channel triggers `runner.py::run_job()`, which builds the initial state and invokes the graph

**Performance Goals**: Slack message within 2 min of JD submission; edit results within 90 sec

**Constraints**: Single user, no auth, local storage only, free-tier AI (Gemini) + observability (Langfuse Hobby 50k obs/month)

## Phase 0: Research

### Resolved Decisions

**LangGraph + langchain-google-genai**
- Decision: LangGraph for workflow (HITL, checkpointing, conditional routing); `langchain-google-genai` for Gemini calls with `.with_structured_output(PydanticModel)`
- Why: Eliminates manual JSON parsing; Gemini SDK transport-layer auto-retry preserved; LangGraph is part of the LangChain ecosystem
- Rejected: Raw `google-generativeai` SDK — no Pydantic structured output, no LangGraph integration

**Langfuse prompt management**
- Decision: `langfuse_client.get_prompt(id)` at runtime (SDK TTL-caches locally); fallback to `harness.json["system_prompt"]` on any exception
- Why: Prompt versioning without code deploys; every trace links to the exact prompt version used
- Rejected: Prompts only in `harness.json` — no versioning, no A/B in Langfuse

**HITL: `interrupt()` + `SqliteSaver`**
- Decision: `interrupt()` in `await_feedback_node`; state in `checkpoints.db`; resumed via `graph.invoke(Command(resume=feedback), config={"configurable": {"thread_id": ...}})`
- Why: Native LangGraph HITL; survives process restarts; no polling or external queue
- Rejected: asyncio queue / Redis — unnecessary infra for a single-user tool

**Slack: Bolt + FastAPI via `SlackRequestHandler`**
- Decision: `SlackRequestHandler(bolt_app)` bridges both under FastAPI at `/slack/events`
- Why: Bolt handles signature verification, event parsing, `ack()`; FastAPI manages HTTP lifecycle
- Rejected: Flask or raw HTTP — loses Bolt's action routing

**Edit routing: `edit_node → notify_node`**
- Decision: Edit applies targeted changes to the current resume then re-posts; does NOT re-run tailor
- Why: Faster, preserves user context; exact match to TRD architecture diagram
- Rejected: `edit → tailor` — would discard the current version entirely

**Two SQLite files**
- Decision: `tailorcv.db` (app data) + `checkpoints.db` (LangGraph SqliteSaver)
- Why: Separate to avoid schema collision; both in same Docker volume

## Phase 1: Design Artifacts

### Data Model

#### SQLite: `tailorcv.db`

```
jobs
  id                INTEGER  PK
  company           TEXT     NOT NULL
  role              TEXT     NOT NULL
  jd_text           TEXT     NOT NULL
  status            TEXT     NOT NULL DEFAULT 'pending'
                    -- pending | approved | rejected | expired | failed
  thread_id         TEXT     NOT NULL UNIQUE  ← uuid4; LangGraph checkpoint key
  slack_ts          TEXT                       ← Slack message ts (for update/thread reply)
  langfuse_trace_id TEXT                       ← links Slack actions to Langfuse trace
  last_activity_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
  created_at        DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP

resume_versions
  id               INTEGER  PK
  job_id           INTEGER  NOT NULL  FK→jobs(id) ON DELETE CASCADE
  version          INTEGER  NOT NULL  ← matches TailorState.iteration
  content          TEXT     NOT NULL  ← JSON: {summary, experience, skills, education}
  keyword_coverage REAL               ← heuristic score at save time
  approved         BOOLEAN  NOT NULL DEFAULT FALSE
  created_at       DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP

Indexes:
  idx_resume_versions_job_id  ON resume_versions(job_id)
  idx_jobs_status_activity    ON jobs(status, last_activity_at)
```

#### LangGraph State: `TailorState` (TypedDict in `graph.py`)

```python
job_id:             int             DB row ID
thread_id:          str             uuid4; checkpointer + DB key
langfuse_trace_id:  str             set once in runner.py::run_job(); shared by all nodes
company:            str             output of analyze_node
role:               str             output of analyze_node
jd_text:            str             raw CLI input
jd_analysis:        Optional[dict]  JDAnalysisOutput.model_dump()
base_resume:        dict            loaded from base_resume.json at startup
tailored_resume:    Optional[dict]  TailoredResumeOutput.model_dump()
slack_ts:           str             set by notify_node (Slack message timestamp)
iteration:          int             0 initial; ++ in edit_node; warn@3, hard stop@4
feedback:           str             "approve"|"reject"|"edit:<text>"|"expired"
status:             str             mirrors jobs.status
error_count:        int             node retry counter; max 2 before hard fail
last_error:         Optional[str]   exception message for Slack/Langfuse error report
scores:             Optional[dict]  ScoreResult.model_dump()
```

#### Pydantic Schemas (`schemas.py`)

```python
class JDAnalysisOutput(BaseModel):
    company: str
    role: str
    required_skills: list[str]
    preferred_skills: list[str]
    keywords: list[str]

class ExperienceEntry(BaseModel):
    company: str; role: str; dates: str; bullets: list[str]

class EducationEntry(BaseModel):
    institution: str; degree: str; year: str

class TailoredResumeOutput(BaseModel):
    summary: str
    experience: list[ExperienceEntry]
    skills: list[str]
    education: list[EducationEntry]

class ScoreResult(BaseModel):
    keyword_coverage: float
    no_hallucination: bool
    edit_fidelity: float | None    # None outside edit loop
    needs_retry: bool              # True when keyword_coverage < 0.5
```

### Interface Contracts

#### Job Entrypoint (`runner.py::run_job()`)

There is no separate CLI entrypoint. A user posts a job description (≥100 chars) as a top-level message in the Slack channel; `slack_handler.py::handle_message` validates the length, posts a "⏳ Tailoring..." placeholder, and runs `run_job(jd_text, graph, loading_ts)` in a background thread:

```
run_job(jd_text, graph, loading_ts="") -> int

1. Load base_resume.json (+ personal.json if present)
2. Create Langfuse trace (trace_id) + Job row (status=pending)
3. Build initial TailorState
4. graph.invoke(initial_state, config={"configurable": {"thread_id": thread_id}})
   — raises GraphInterrupt when paused at await_feedback_node (expected)
```

#### Slack Block Kit (`contracts/slack-blocks.md`)

Review message block order:
1. `header`: `"{company} – {role}"`
2. `section`: resume summary
3. `section`: scores (`keyword_coverage`, `no_hallucination`)
4. `context` (only when `iteration == 3`): `"⚠️ This is your last edit"`
5. `actions`:
   - `approve_resume` — value: `"{job_id}"`, style: `"primary"`
   - `edit_resume` — value: `"{job_id}"` (omitted when `iteration >= 4`)
   - `reject_resume` — value: `"{job_id}"`, style: `"danger"`

Action → graph feedback mapping:

| `action_id`       | Handler            | `Command(resume=...)`     |
|-------------------|--------------------|---------------------------|
| `approve_resume`  | `handle_approve`   | `"approve"`               |
| `edit_resume`     | `handle_edit`      | waits for thread reply    |
| `reject_resume`   | `handle_reject`    | `"reject"`                |
| thread `message`  | `handle_message`   | `"edit:<user text>"`      |
| startup sweep     | `stale_sweep()`    | `"expired"`               |

#### Langfuse Trace Schema (`contracts/langfuse-scores.md`)

One trace per job. Scores attached:

| Score name         | Type  | Attached when                     | Range       |
|--------------------|-------|-----------------------------------|-------------|
| `keyword_coverage` | float | After `tailor_node`, `edit_node`  | 0.0 – 1.0   |
| `no_hallucination` | float | After `tailor_node`, `edit_node`  | 0.0 or 1.0  |
| `edit_fidelity`    | float | `edit_node` only                  | 0.0 – 1.0   |
| `user_approval`    | float | Approve (1.0) or Reject (0.0)     | 0.0 or 1.0  |

Langfuse prompt IDs must match `harness.json` `id` fields: `analyze-jd`, `tailor-resume`, `edit-resume`.

Quality scoring (LLM-as-a-judge) is configured natively in Langfuse against the `tailored-resume` trace observation (see `contracts/langfuse-scores.md`) — not computed by application code.

## Key Integration Points

**`slack_handler` → LangGraph resume**
1. Button click → Bolt action handler `ack()` immediately
2. `job_id` from `body["actions"][0]["value"]` → DB lookup for `thread_id`
3. `graph.invoke(Command(resume=feedback), config={"configurable": {"thread_id": job.thread_id}})`
4. `SqliteSaver` restores `TailorState` from `checkpoints.db`; graph resumes at `await_feedback_node`

**Edit text capture (two steps)**
1. `edit_resume` click → bot posts thread reply "What would you like to change?"
2. `thread_ts` stored in `_pending_edit_registry: dict[str, int]` (thread_ts → job_id)
3. `@bolt_app.event("message")` fires on user's reply; pops registry; calls `graph.invoke(Command(resume=f"edit:{text}"), ...)`

**Langfuse trace shared across all nodes**
- Created once in `runner.py::run_job()`: `trace_id = langfuse.create_trace_id()`, then `start_observation(trace_context={"trace_id": trace_id}, name="resume-tailoring", input={...}).end()`
- `trace_id` stored in `TailorState["langfuse_trace_id"]` and `jobs.langfuse_trace_id`
- Every scoring node calls `get_langfuse().create_score(trace_id=state["langfuse_trace_id"], name=..., value=...)`
- `slack_handler` reads `job.langfuse_trace_id` from DB to attach `user_approval`

**Harness prompt loading**
- `load_harnesses("harness.json")` once at startup → dict keyed by `id`
- `get_prompt(id)`: `langfuse_client.get_prompt(id)` (SDK TTL-cached) → fallback to `harness["system_prompt"]`
- `get_llm(id)`: returns `ChatGoogleGenerativeAI(model=harness["model_name"], ...)` — stateless, cheap per call

**Startup sequence in `slack_handler.py::_lifespan()`**
1. `load_harnesses("harness.json")`
2. `init_db()` — creates tables if not exist
3. build checkpointer + graph
4. `stale_sweep(get_slack(), graph)` — one-shot expire jobs > 24h

**Per-job sequence in `runner.py::run_job()`** (triggered by `handle_message` on a new top-level Slack message)
1. parse JD text from the Slack message (`handle_message` validates length first)
2. load `base_resume.json` (+ `personal.json` if present)
3. create Langfuse trace; store `trace_id` in `Job.langfuse_trace_id`
4. create `Job` row; generate `thread_id = str(uuid4())`
5. `graph.invoke(initial_state, config={"configurable": {"thread_id": thread_id}})` — blocks at `interrupt()`

```
Step 6a  nodes/analyze.py
Step 6b  nodes/tailor.py            ← calls score inline; silent retry on kw < 0.5
Step 6c  nodes/notify.py            ← Block Kit post/update; iteration warning/guard
Step 6d  nodes/await_feedback.py    ← interrupt() only; hard-stop at iteration >= 4 enforced by route_feedback() (step 5)
Step 6e  nodes/edit.py              ← edit harness + score + iteration++ + ResumeVersion save

Step 6g  nodes/finalize.py          ← WeasyPrint PDF + DB approved + user_approval score
Step 7   slack_handler.py           ← FastAPI + Bolt; action handlers → graph.invoke(Command)
Step 8   runner.py                  ← run_job() + stale_sweep(); startup sequence in slack_handler lifespan
Step 9   evals/dataset.json + evals/run_evals.py
Step 10  Dockerfile + docker-compose.yml
```
